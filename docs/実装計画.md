# 実装計画

## 技術選定
| 項目 | 選定 | 理由 |
|---|---|---|
| 言語 | HTML / CSS / JavaScript | ブラウザネイティブ、PC・スマホ両対応 |
| フレームワーク | Vue 3（CDN版） | 軽量、ビルド不要、学習容易 |
| スタイル | Tailwind CSS（CDN）+ 素のCSS | 速い、レスポンシブ容易 |
| データ永続化 | localStorage（基本）+ Firebase Firestore（同期） | 無料、設定容易 |
| ホスティング | GitHub Pages | 無料、URLアクセス可 |
| ポケモン基礎データ | PokeAPI由来のローカルJSON（同梱） | オフラインで動く、高速 |

## フェーズ分け

### Phase 1: コア機能（オフライン完結）
**目標**: PCで自分のパーティを登録して選出シミュレートが動く
- [ ] プロジェクト構造構築
- [ ] pokemon_champions.json 整備
- [ ] core/kana.js: ひらがな正規化
- [ ] core/search.js: 予測入力インデックス
- [ ] core/storage.js: localStorage 永続化
- [ ] core/score.js: スコアリング実装
- [ ] views/TypeEdit: 型編集画面
- [ ] views/TypeList: 型一覧画面
- [ ] views/PartyEdit/PartyList: パーティ管理
- [ ] views/Simulator: 選出シミュレータ
- [ ] views/Home: ホーム

### Phase 2: 利便性向上
- [ ] 略称辞書の編集UI
- [ ] スコア重み設定

- [ ] 候補のソート/絞り込み調整
- [ ] エラーハンドリング・バリデーション

### Phase 3: スマホ対応・同期
- [ ] レスポンシブCSS調整
- [ ] スマホUI最適化（ボタン大きく、入力しやすく）
- [ ] Firebase 同期機能
- [ ] ログイン機能（Google認証）

### Phase 4: メンテナンス機能
- [ ] 使用可能ポケモンのインポート/エクスポート
- [ ] 全データのバックアップ/リストア
- [ ] PWA化（ホーム画面追加、オフライン強化）

## タスク優先度
最初に動くものを作るために、Phase 1の以下を優先:
1. データ構造とstorage
2. score.js（ロジック検証用）
3. Simulator画面（手動データで動作確認）
4. TypeEdit/PartyEdit（データ登録できるように）
5. その他画面

## テスト方針
- core/score.js は単体テスト必須（スコア計算ロジックの正しさ）
- ブラウザでの手動テストで画面動作を確認
- 複数の相手パーティパターンを用意して挙動を検証
