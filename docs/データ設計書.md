## 2. エンティティ定義

### Pokemon（マスタ・読み取り専用）
```ts
{
  id: number,         // 全国図鑑番号
  en: string,         // 英語名
  ja: string,         // 日本語名（カタカナ）
}
```
※ ひらがな正規化は実行時に自動生成。

### Type（ポケモンの型・ユーザー作成）
```ts
{
  id: string,                    // UUID
  pokemonId: number,             // → Pokemon.id
  nickname: string,              // 型名/メモ（任意）
  item: string,                  // 持ち物（フリー）
  moves: [string,string,string,string], // 技4つ（フリー）
  evs: {
    hp: number,   // 0-32
    atk: number,  // 0-32
    def: number,  // 0-32
    spa: number,  // 0-32
    spd: number,  // 0-32
    spe: number   // 0-32
  },                             // 合計 ≤ 66
  nature: string,                // 性格名
  isMega: boolean,               // メガシンカフラグ
  isPriority: boolean,           // 優先ポケモンフラグ
  matchups: {                    // 相手ポケモンID → 記号
    [pokemonId: number]: "○" | "△" | "×"
  },
  createdAt: number,
  updatedAt: number
}
```

### Party
```ts
{
  id: string,                    // UUID
  name: string,                  // パーティ名
  typeIds: [string,string,string,string,string,string],
  createdAt: number,
  updatedAt: number
}
```

### Settings
```ts
{
  weights: {
    coverO: 10,         // ○でカバー
    coverDouble: 6,     // △2匹でカバー
    coverSingle: 3,     // △1匹のみ
    detailO: 2,         // 詳細点 ○
    detailX: -2,        // 詳細点 ×
    megaBonus: 5,       // メガシンカボーナス
    priorityBonus: 5    // 優先ポケモンボーナス
  },
  showOnlyUsable: true, // 使用可能のみ表示
  defaultPartyId: string | null,
  candidateCount: 5     // 表示する候補数
}
```

### Abbreviation
```ts
{
  short: string,        // 略称（例: "ガブ"）
  pokemonId: number     // → Pokemon.id
}
```

### UsableEntry
```ts
{
  pokemonId: number,
  usable: boolean
}
```

## 3. 検索インデックス（メモリ）

起動時に以下を構築:

```ts
SearchIndex = {
  byJa:    Map<string, Pokemon>,    // "ガブリアス" → P
  byJaHira: Map<string, Pokemon>,   // "がぶりあす" → P
  byEn:    Map<string, Pokemon>,    // "garchomp" → P (lower)
  byShort: Map<string, Pokemon>,    // "ガブ" → P
  byId:    Map<number, Pokemon>     // 445 → P
}
```

予測入力は前方一致＋部分一致を全インデックスに対して実行し、重複を除いて返す。

## 4. ひらがな正規化
```js
katakanaToHiragana(s) =>
  s.replace(/[\u30A1-\u30F6]/g, c => 
    String.fromCharCode(c.charCodeAt(0) - 0x60))
```
逆変換も同様。検索クエリ側で両方ヒットさせる。

## 5. データのインポート/エクスポート
- JSONエクスポート: 全ユーザーデータを1ファイルでバックアップ
- JSONインポート: 上記の復元
- 使用可能リストのテキストインポート: 1行1ポケモン名（英/日）、上書き/追加モードあり
- 使用可能リストのJSONインポート: Champions console抽出形式 `[{dex, form, name}]`、上書き/追加モードあり

## 6. 追記（2026-04-12）

### ポケモンマスタの拡張
- `data/pokemon_champions.json` は廃止し、`js/data.js` に1052エントリをインライン化
- 内訳: 通常種1025匹 + リージョンフォーム等27エントリ
- フォームバリアントはPokeAPIの高ID（10000番台）をそのまま使用
- 日本語名はPokeAPIから取得、一部手動修正（カミツオロチ等）

### enabledPokemon（使用可能ポケモン）
```ts
// localStorage キー: poke.enabledPokemon
// 型: number[]  (Pokemon.id の配列)
// デフォルト: DEFAULT_ENABLED_IDS (213匹、2026-04-12時点)
```
- `App.Store.getEnabledIds()`: 保存済みIDリスト取得（未保存ならデフォルト返却）
- `App.Store.saveEnabledIds(ids)`: 保存
- `App.Store.getEnabledPokemon()`: IDリストからPokeモンオブジェクト配列に変換
- `App.Store.DEFAULT_ENABLED_IDS`: デフォルト213匹のIDリスト

### USAGE_RANK
```js
// js/data.js 末尾に定義
App.USAGE_RANK = { pokemonId: rank, ... }
// 例: { 445: 1, 6: 2, 730: 3, ... }  (ALTEMA TOP50)
```

### Champions console抽出形式
```json
[
  { "dex": 445, "form": 0, "name": "ガブリアス" },
  { "dex": 26,  "form": 1, "name": "ライチュウ (アローラのすがた)" }
]
```
- `dex`: 全国図鑑番号
- `form`: フォーム番号（0=通常、1+=リージョン等）
- `name`: 日本語名（マスタ照合に使用）

抽出コード: `enable_poke/ポケモン取得用コード.txt`
