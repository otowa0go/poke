## 2. エンティティ定義

### Pokemon（マスタ・読み取り専用）
```ts
{
  id: number,         // 全国図鑑番号
  en: string,         // 英語名
  ja: string,         // 日本語名（カタカナ）
}
```
※ ひらがな正規化は実行時に自動生成。

### Type（ポケモンの型・ユーザー作成）
```ts
{
  id: string,                    // UUID
  pokemonId: number,             // → Pokemon.id
  nickname: string,              // 型名/メモ（任意）
  item: string,                  // 持ち物（フリー）
  moves: [string,string,string,string], // 技4つ（フリー）
  evs: {
    hp: number,   // 0-32
    atk: number,  // 0-32
    def: number,  // 0-32
    spa: number,  // 0-32
    spd: number,  // 0-32
    spe: number   // 0-32
  },                             // 合計 ≤ 66
  nature: string,                // 性格名
  isMega: boolean,               // メガシンカフラグ
  isPriority: boolean,           // 優先ポケモンフラグ
  matchups: {                    // 相手ポケモンID → 記号
    [pokemonId: number]: "○" | "△" | "×"
  },
  createdAt: number,
  updatedAt: number
}
```

### Party
```ts
{
  id: string,                    // UUID
  name: string,                  // パーティ名
  typeIds: [string,string,string,string,string,string],
  createdAt: number,
  updatedAt: number
}
```

### Settings
```ts
{
  weights: {
    coverO: 10,         // ○でカバー
    coverDouble: 6,     // △2匹でカバー
    coverSingle: 3,     // △1匹のみ
    detailO: 2,         // 詳細点 ○
    detailX: -2,        // 詳細点 ×
    megaBonus: 5,       // メガシンカボーナス
    priorityBonus: 5    // 優先ポケモンボーナス
  },
  showOnlyUsable: true, // 使用可能のみ表示
  defaultPartyId: string | null,
  candidateCount: 5     // 表示する候補数
}
```

### Abbreviation
```ts
{
  short: string,        // 略称（例: "ガブ"）
  pokemonId: number     // → Pokemon.id
}
```

### UsableEntry
```ts
{
  pokemonId: number,
  usable: boolean
}
```

## 3. 検索インデックス（メモリ）

起動時に以下を構築:

```ts
SearchIndex = {
  byJa:    Map<string, Pokemon>,    // "ガブリアス" → P
  byJaHira: Map<string, Pokemon>,   // "がぶりあす" → P
  byEn:    Map<string, Pokemon>,    // "garchomp" → P (lower)
  byShort: Map<string, Pokemon>,    // "ガブ" → P
  byId:    Map<number, Pokemon>     // 445 → P
}
```

予測入力は前方一致＋部分一致を全インデックスに対して実行し、重複を除いて返す。

## 4. ひらがな正規化
```js
katakanaToHiragana(s) =>
  s.replace(/[\u30A1-\u30F6]/g, c => 
    String.fromCharCode(c.charCodeAt(0) - 0x60))
```
逆変換も同様。検索クエリ側で両方ヒットさせる。

## 5. データのインポート/エクスポート
- JSONエクスポート: 全ユーザーデータを1ファイルでバックアップ
- JSONインポート: 上記の復元
- 使用可能リストのテキストインポート: 1行1ポケモン名（英/日）
