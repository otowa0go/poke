# 開発メモ

## 仕様確定済み事項
- 実装方式: Webアプリ + クラウド同期
- スコア計算: カバレッジ点（10/6/3）+ 詳細点（+2/-2）+ ボーナス（メガ/優先）
- 重みは設定画面で変更可能
- 「全カバー警告」ではなく「重い相手」として該当ポケモンを表示
- △の扱い: △2匹で部分カバー（+6）、△1匹のみ（+3）
- ポケモンDB: PokeAPI由来のローカルJSON同梱
- 使用可能ポケモン: 要望.txtの186匹を初期値、ユーザー編集可
- ひらがな対応: 自動（カタカナ⇔ひらがな相互変換）
- 略称: 初期辞書同梱、ユーザー追加可
- 努力値: チャンピオンズ仕様（各0〜32 / 合計66）
- 持ち物・技: フリー入力（マスタなし）
- 同名ポケモンの複数登録: 内部IDで区別、表示は「ポケモン名 (型名)」

## 既知の課題・注意点

### 1. ポケモンチャンピオンズ仕様の不確実性
- 公式の使用可能ポケモンの機械可読リストが現状なし
  → 要望.txtの186匹を初期値とし、インポート機能で更新可能にする
- 性格・技・持ち物のチャンピオンズ仕様は要確認
  → 持ち物・技はフリー入力で回避、性格は従来と同じ25種を仮置き

### 2. 日本語名・図鑑番号の正確性
- pokemon_champions.json のポケモン日本語名と図鑑番号は AI が生成しているため
  誤りが含まれる可能性あり
  → 使用前に目視確認、誤りがあれば修正

### 3. メガシンカ
- どのポケモンがメガシンカ可能かはユーザーが型ごとに「メガシンカフラグ」で指定
- メガシンカ後のポケモンを別の型として登録する運用も可能

### 4. 略称辞書の初期収録
- 私が知っている範囲の略称を `data/abbreviations.json` に収録
- 不足分はユーザーが設定画面から追加

## 設計上の判断記録
| 日付 | 判断 | 理由 |
